Welcome!

Use next endpoints for:

  • @ GET /api/contacts

    • Gets nothing
    • Returns an array of all contacts in json format with status 200

  • @ GET /api/contacts/:contactId

    • Doesn't get body
    • Gets the `contactId` parameter of type ObjectId
    • If id is not valid ObjectId - returns error "message":"Your id is not valid" with status 400
    • If there is such an id, it returns the contact object in json format with the status 200
    • if there is no such id, it returns json with the key `"message":"Not found"` and 404 status

  • @ POST /api/contacts

    • Gets a body in the format `{name, email, phone}`, all fields are required
    • If there are missing required fields in the body, it returns an joi validation error object with a status of 400
    • If everything is fine with body, saves the new contact to the database
    • returns an object with an added id `{_id, name, email, phone}` and status 201

  • @ DELETE /api/contacts/:contactId

    • Doesn't get body
    • Gets the `contactId` parameter of type ObjectId
    • If id is not valid ObjectId - returns error "message":"Your id is not valid" with status 400
    • if there is such an id, it returns json of the format `{"message":"Contact deleted"}` and status 200
    • if there is no such id, it returns json with the key `" message ":" Not found "` and 404 status

  • @ PATCH /api/contacts/:contactId

    • Gets the `contactId` parameter of type ObjectId
    • If id is not valid ObjectId - returns error "message":"Your id is not valid" with status 400
    • Receives body in json-format with updating of any fields `name, email and phone`, one of the listed is required
    • If there are missing required fields or wrong fields in the body, it returns an joi validation error object with a status of 400
    • If everything is fine with body, updates the contact in the database
    • Returns the updated contact object and status 200.
    • if there is no such id, it returns json with the key `" message ":" Not found "` and 404 status
